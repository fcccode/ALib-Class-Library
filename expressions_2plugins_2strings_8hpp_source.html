109  * # Notes And Hints #
110  *
111  * ### Compile-Time Invokable: ###
112  * All callback functions are defined compile-time invokable. This means that redundancies in
113  * string expressions emerging from operations on constant strings are optimized (pruned) by the
114  * compiler.
115  *
116  * ### Underlying %String %Types: ###
117  * While all string manipulation is based on \alib classes \alib{strings,StringBase,String},
118  * \alib{strings,AStringBase,AString} and \alib{strings,SubstringBase,Substring}, due to the
119  * "harmonizing" way that string classes become boxed, none of these types appears as an expression
120  * result type. This means that custom expression functions can unbox strings that have been created
121  * by callback functions of this plug-in to their own custom string type (e.g. \c std::string,
122  * \c QString, or whatever), and, the other way round, results of custom callback functions that
123  * returned custom strings objects, can seamlessly be used by functions and operators defined here.
124  *
125  * Details of this are explained in
126  * \ref alib_namespace_boxing_basics_nonbij "ALib Boxing: 3.5 Non-Bijective Type Mapping".
127  *
128  *
129  * ### Determine A String's Length ###
130  *
131  * The length of the string can be determined with function \b %Length, which is defined with
132  * compiler plugin \alib{expressions::plugins,Arithmetics} (because it works on all arrays).
133  *
134  * ### Test For Empty %Strings: ###
135  *
136  * Unary operator \alib{expressions::DefaultUnaryOperators,BoolNot} (<c>'!'</c>) may be used to check if a
137  * string is not empty. The operator returns \c true if the string is empty and \c false otherwise.<br>
138  *
139  * Consequently, to test for non-empty strings, <c>'!!'</c> may be written. The outer not
140  * operation is then a not operator on the boolean result of the inner.<br>
141  * Alternatively, binary operators (see below) against an emtpy string can be used:
142  *
143  * myIndentifier == ""
144  * myIndentifier != ""
145  *
146  * As a result, this expressions is a tautology:
147  *
148  * !myIdentifier == (myIndentifier == "")
149  *
150  * Finally with strings, the built-in implementation of the
151  * \alib{expressions,plugins::ElvisOperator,elvis operator} is quite useful to avoid empty
152  * strings. The expression:
153  *
154  * myIdentifier ?: "Default"
155  *
156  * returns "Default" if \e myIdentifier evaluates to an empty string.
157  *
158  *
159  * ### Case conversion: ###
160  * Unary operators \alib{expressions::DefaultUnaryOperators,Positive} <c>'+'</c> and
161  * \alib{expressions::DefaultUnaryOperators,Negative} <c>'-'</c> are alias to functions \b %ToUpper and
162  * \b %ToLower.
163  * Hence, the expressions
164  *
165  * ToUpper( "Hello " ) + ToLower( "World" )
166  * +"Hello " + -"World"
167  * are equivalent.
168  *
169  *
170  * ### Concatenation: ###
171  * Binary operator <c>'+'</c> is compiled by this plug-in if one of the arguments (or both) is of
172  * string type. If one argument is not of string type, it becomes converted. For the conversion,
173  * the following rules apply:
174  *
175  * - Types \alib{expressions::Types,Integer} and \alib{expressions::Types,Float} are converted
176  * to string types using object \alib{strings::format::FormatterBase,DefaultNumberFormat} of
177  * field \alib{expressions,Scope::Formatter}.
178  * With this, detailed options for the number output format is available.
179  * Even more options are available when using expression function \b %Format instead of
180  * the <c>'+'</c> operator.
181  *
182  * <p>
183  * - Any other type is converted to a string representation using \alib \b %Boxing interface
184  * \alib{strings::boxing,IApply}. For information on how to implement this for custom
185  * types, consult the documentation of module \alibmod_boxing. The following provides a quick
186  * sample snippet, taken from the unit tests.
187  *
188  * 1. The custom type:
189  * \snippet "ut_alib_expr_basics.cpp" DOX_ALIB_EXPRESSIONS_STRINGOPS_IAPPLY_1
190  *
191  * 2. Definition of interface \b %IApply for type <b>MyType</b>:
192  * \snippet "ut_alib_expr_basics.cpp" DOX_ALIB_EXPRESSIONS_STRINGOPS_IAPPLY_2
193  *
194  * 3. In the bootstrap section of an application, the interface has to be registered with \alibmod_nolink_boxing:
195  * \snippet "ut_alib_expr_basics.cpp" DOX_ALIB_EXPRESSIONS_STRINGOPS_IAPPLY_3
196  *
197  * With this short setup code, objects of type \e MyType, which might be returned by custom
198  * identifiers, can be added to strings in expressions!
199  *
200  * <p>
201  * - With type \alib{expressions::Types,Boolean}, the \alib{lang,Resources,resourced}
202  * string values generally used with boxing interface \alib{strings::boxing,IApply} defined for
203  * C++ type \c bool are used.
204  * These string resources default to \b "true" and \b "false". While resources can be customized,
205  * such customization has process-wide effect.<br>
206  * To express other values, \ref alib_expressions_builtin_ternary "conditional operator"
207  * (<c>Q ? T : F</c>) can be used by the end-user:
208  *
209  * "Is true: " + {boolean expression} // Uses the global resourced strings
210  * "Is true: " + ( {boolean expression} ? "Yes" : "No" ) // Custom conversion per expression.
211  *
212  * ### Comparison: ###
213  * Operators <c>'=='</c>, <c>'!='</c>, <c>'<'</c>, <c>'<='</c>, <c>'>'</c> and <c>'>='</c>
214  * perform comparisons between two string operands.
215  * Internally, to perform the comparison, methods \alib{strings,StringBase::Equals,String::Equals} and
216  * \alib{strings,StringBase::CompareTo,String::CompareTo} are used.
217  *
218  * Case insensitive comparison can be performed by converting the operand(s) to upper case
219  * using expression function \b %ToUpper or its "alias operator", unary <c>'+'</c>.
220  * A more effective way is to use overloaded function \b %Compare that accepts an optional
221  * third operator of type \b %Boolean, which if \c true is given performs a case insensitive
222  * search without performing string conversions.
223  *
224  *
225  * ### Formatting: ###
226  * Function <b>Format(String, ...)</b> offers full featured string formatting in expressions.
227  * Being based on classes of underlying module \alibmod_strings, there is a choice between
228  * \alib{strings::format,FormatterPythonStyleBase,python style} or
229  * \alib{strings::format,FormatterJavaStyleBase,Java/printf style} format strings. It is even possible
230  * to allow both formats, of-course not within the same string, but within different
231  * evaluations of the same expression!
232  *
233  * Please consult the documentation of classes
234  * - \alib{strings::format,FormatterBase,Formatter},
235  * - \alib{strings::format,FormatterBase,Formatter},
236  * - \alib{strings::format,FormatterPythonStyleBase,FormatterPythonStyle} and
237  * - \alib{strings::format,FormatterJavaStyleBase,FormatterJavaStyle} for details.
238  *
239  *
240  * ### Wildcard Match: ###
241  * Wildcard match is implemented with expression function \b %WildcardMatch, respectively its "alias
242  * operator" <c>'*'</c>.
243  * The first (left-hand side) argument is the
244  * string that is searched, the second (right-hand side) argument is the pattern string to be
245  * matched.
246  *
247  * Wildcards characters are (<c>'*'</c>) and (<c>'?'</c>). For example, expressions
248  *
249  * WildcardMatch("This is ALib Expressions", "*A?ib*")
250  * "This is ALib Expressions" * "*A?ib*"
251  *
252  * are equivalent and return boolean \c true.
253  *
254  * For case insensitive search, the both strings are to be converted to upper case, as in
255  * expression:
256  *
257  * +filename * "*.JPG"
258  *
259  * \note
260  * A more performant alternative for the latest sample expression is:
261  *
262  * EndsWith( filename, ".jpg", true )
263  *
264  * ### Regular %Expression Match: ###
265  * Regular expression match is implemented with expression function \b %RegexMatch, respectively its
266  * "alias operator" <c>'\%'</c>.
267  * The first (left-hand side) argument is the
268  * string that is searched, the second (right-hand side) argument is the pattern string to be matched.
269  *
270  * The regular expression syntax is compatible with
271  * [Perl Regular Expressions](http://perldoc.perl.org/perlretut.html).
272  * The feature is implemented with using [boost::regex library](http://www.boost.org).
273  *
274  * \attention
275  * The regular expression features of \alib and therefore of this compiler plug-in, are only
276  * available with the definition of code selection symbol \ref ALIB_FEAT_BOOST_REGEX,
277  * which is controlled by using compiler symbols #ALIB_FEAT_BOOST_REGEX_ON and
278  * #ALIB_FEAT_BOOST_REGEX_OFF (the default).
279  *
