26 /** ************************************************************************************************
27  * This interface class represents "plug-ins" which are attachable to container class
28  * \alib{expressions,Compiler}. The compiler uses the plug-ins to <em>"compile"</em> single
29  * nodes of <em>abstract syntax trees</em>, which are the intermediate, internal result of parsing
30  * expression strings.
31  *
32  * In depth information on how this class is used is given in the
33  * \ref aworx::lib::expressions "User Manual And Tutorial" of module \alibmod_nolink_expressions.
34  *
35  * The overloaded methods #TryCompilation are not abstract, but have a default implementation
36  * that return constant \c false. A plug-in derived from this class needs to override only those
37  * methods that represent syntax elements of an expression, that it volunteers to take
38  * responsibility for.
39  *
40  * The overloaded methods have one input/output parameter which is of one of the following types:
41  * - \alib{expressions::CompilerPlugin,CIFunction},
42  * - \alib{expressions::CompilerPlugin,CIUnaryOp},
43  * - \alib{expressions::CompilerPlugin,CIBinaryOp} or
44  * - \alib{expressions::CompilerPlugin,CIBinaryAutoCast}.
45  *
46  * These inner types of this struct are derived from likewise inner struct
47  * \alib{expressions::CompilerPlugin,CompilationInfo}, which provides all output members and
48  * those input members that are common between the four descendants. In other words, the variants
49  * of the struct provide detail information on the node that is actually compiled.
50  *
51  * When overriding one of the overloaded methods #TryCompilation, it has to be checked if
52  * the custom plug-in is responsible for the permutation of given parameter types which are passed.
53  * If so, the actual 'compilation' is performed, by filling the result members of the given
54  * struct and by returning \c true. Each overloaded
55  * method provides a suitable return type, which are defined as inner types of this class.
56  *
57  * The output information comprises a \ref alib_expressions_prereq_sb "sample box" that
58  * determines the result type of the native C++ function that is later invoked when the expression
59  * is evaluated against a scope.
60  * Together with this type information, the pointer to the callback function itself needs to be
61  * set in the given struct.
62  *
63  * Alternatively, if a constant result is to be returned, the pointer to the callback function
64  * might be kept \e nulled. In this case, the \b %Box object that represents the result type
65  * of the callback function is used to provide the constant value, instead of just a sample value.
66  *
67  * There are two possibilities, why a compiler plug-in might return a constant value, instead of
68  * a function:
69  * 1. \e Identifiers (parameterless functions) might just represent constant values. (For example
70  * identifiers \c "True" or \c "Monday" do that, while \c "Today" returns a callback function.).
71  * 2. The compiler provides information about whether the given parameters are constant values.
72  * In this case, a compiler plug-in may decide to evaluate the result of the function at
73  * compile time. This is often, but not always possible and depends largely on the fact
74  * if scope information is used with the function as well. <br>
75  * For more details on this topic see manual section
76  * \ref alib_expressions_details_optimizations "11.5 Optimizations".
77  *
78  * \attention
79  * If constant data is returned, it has to be assured, that the contents of the returned
80  * \b %Box remains valid during the lifecycle of the expression. This is assured for all
81  * \ref alib_namespace_boxing_types_ftypes "boxed C++ fundamental types". For custom types
82  * it depends on where the constant value is received from and how boxing is performed.
83  * (By default, custom types are boxed as pointers to the assigned object.)<br>
84  * Field \alib{expressions::CompilerPlugin,CompilationInfo::CompileTimeScope}
85  * is to be used for compile-time allocations.
86  *
87  * The plug-ins are attached to the \b %Compiler using a dedicated prioritization defined
88  * with (arithmetical) enum type \alib{expressions,CompilePriorities}.
89  * This means, that if a higher prioritized plug-in already compiles a certain
90  * permutation of expression node type and argument types, then a lower prioritized plug-in is not
91  * even asked to do so. With this concept, <b>ALib Expressions</b> provides a lot of flexibility:
92  * the built-in operators and identifiers can be unplugged completely or just sparsely "superseded"
93  * in certain aspects by adding a custom \b %CompilerPlugin with a higher priority than that of
94  * the built-in one.
95  *
96  * \note
97  * Instead of deriving from this struct, it is recommended to consider convenience struct
98  * \alib{expressions,plugins::Calculus} instead.
99  *
100  * \note
101  * More information on this topic is given in manual section
102  * \ref alib_expressions_cpcc_calculus "5.4 Class Calculus"
103  **************************************************************************************************/

116  /** ********************************************************************************************
117  * Public inner base struct which provides input and output information for compiling
118  * single entities (nodes of the \e AST) of a parsed expression.
119  *
120  * Descendant types
121  * - \alib{expressions::CompilerPlugin,CIFunction},
122  * - \alib{expressions::CompilerPlugin,CIUnaryOp} and
123  * - \alib{expressions::CompilerPlugin,CIBinaryOp}
124  *
125  * extend this struct with input information, specific to the type of <em>AST</em>-node
126  * currently compiled. Special, descendant type
127  * - \alib{expressions::CompilerPlugin,CIBinaryAutoCast}
128  *
129  * in addition adds specific output members.
130  *
131  * Together, these four descendants comprise the parameters of the four overloaded methods
132  * \alib{expressions::CompilerPlugin,TryCompilation}.
133  **********************************************************************************************/
134  struct CompilationInfo
135  {
136  /**
137  * The scope found here is the the same object that is passed to method
138  * \alib{expressions,Compiler::Compile}, which internally invokes the overloaded
139  * methods #TryCompilation, that receive an object of this type.
140  *
141  * If a compiled expression syntax element evaluates to a constant stored in
142  * #TypeOrValue it has to be assured that the boxed data is to available during the
143  * lifecycle of the expression.
144  *
145  * To allocate custom compilation data, a custom, derived type, might for example be
146  * extended with simple\c std::vector of pointers to the created objects.
147  * (Attention: Vectors of value-types must not be used, as with their growth, the objects
148  * get moved within the heap memory!).<br>.
149  * Any data allocated, is to be deleted in the virtual destructor of the custom scope type.
150  */
151  Scope& CompileTimeScope;
152 
153  /**
154  * Input: A start iterator to \ref alib_expressions_prereq_sb "sample boxes"
155  * that provide the argument types to search a native callback function for.<br>
156  * In case that descendant classes denote that arguments found in this iterator are
157  * constant, then the boxes are not just sample boxes, but contain the constant values.
158  */
159  ArgIterator ArgsBegin;
160 
161  /**
162  * Input: The end iterator to #ArgsBegin. The expression <c>ArgsEnd - ArgsBegin</c> may be
163  * used to receive the number of arguments provided.
164  */
165  ArgIterator ArgsEnd;
166 
167  #if ALIB_DEBUG
168  /**
169  * Output: The C++ name of the callback function. This field is available only in debug
170  * compilations of the library. Hence, setting it must be performed with preprocessor
171  * conditionals.
172  */
173  const char* DbgCallBackName = nullptr;
174  #endif
175 
176  /**
177  * Output: The native C++ callback function to be set by one of the plug-ins.
178  */
179  CallbackDecl Callback = nullptr;
180 
181  /**
182  * Output: Specifies the return type of #Callback, respectively, as the name indicates, the
183  * result value in case of constant results.<br>
184  * Note that in case of constant compile-time values, it might be necessary to allocate
185  * compile-time memory for the values. For this, field #CompileTimeScope is to be used.
186  */
187  Box TypeOrValue = nullptr;
188 
189 
190  /**
191  * Constructor.
192  * @param scope The scope usable for allocation of constant values (compile time
193  * allocations).
194  */
195  CompilationInfo( Scope& scope )
196  : CompileTimeScope( scope )
197  , ArgsBegin ( scope.Stack.begin() )
198  , ArgsEnd ( scope.Stack.end() )
199  {}

310  /** ********************************************************************************************
311  * Info struct for compiling expression a binary operation.
312  * This struct is used with method \ref TryCompilation(CIBinaryOp&) to provide information
313  * to derived compiler plug-ins, as well as to receive information back.
314  *
315  * Information about whether the arguments are constants is separately given for \e lhs and
316  * \e rhs with fields #LhsIsConst and #RhsIsConst.
317  *
318  * If both flags evaluate to \c true, operator callback functions that do not use context
319  * information from the scope (or otherwise rely on external or random data), should calculate
320  * the - then constant - result at compile time and return a constant value instead of the
321  * callback method.
322  *
323  * If exactly one of the argument is constant, then some operators might detect further
324  * possibilities of optimization. Such optimization is operator-specific but goes along the
325  * lines of the following samples:
326  *
327  * term + 0 -> term
328  * term - 0 -> term
329  * term * 0 -> 0
330  * term * 1 -> term
331  * term / 1 -> term
332  * term && false -> false
333  * term && true -> term
334  * term || true -> true
335  * term || false -> term
336  *
337  * These cases have in common that the result of the operator is either a constant
338  * or equals the non-constant argument.
339  *
340  * In both cases inherited field \c %Callback is to be left \c nullptr. If the result is
341  * constant, inherited field \b %TypeOrValue is to be set to the constant value.
342  * In the second case that the non-constant argument equals the result, this can be indicated
343  * by setting field #NonConstArgIsResult to \c true. In this case, the compiler will choose
344  * the non-constant argument and drop the other.
345  *
346  * \note
347  * Class \alib{expressions::plugins,Calculus} which specializes this plug-in class,
348  * provides a convenient way to define the optimization rules described here, along with
349  * its mechanics to support binary operator compilation.
350  **********************************************************************************************/
351  struct CIBinaryOp : public CompilationInfo
352  {
353  String& Operator; ///< Input/Output: The binary operation symbol.
354 
355  /// Input: Denotes if the lhs-argument is a constant value.
356  bool LhsIsConst;
357 
358  /// Input: Denotes if the rhs-argument is a constant value.
359  bool RhsIsConst;
360 
361  /// Output: Used with optimization, see this struct's documentation for more information.
362  bool NonConstArgIsResult;
363 
364 
365  /**
366  * Constructor.
367  * @param scope Passed to parent.
368  * @param op Stored in #Operator.
369  * @param lhsIsConst Stored in #LhsIsConst.
370  * @param rhsIsConst Stored in #RhsIsConst.
371  */
372  CIBinaryOp( Scope& scope, String& op, bool lhsIsConst, bool rhsIsConst)
373  : CompilationInfo ( scope )
374  , Operator ( op )
375  , LhsIsConst ( lhsIsConst )
376  , RhsIsConst ( rhsIsConst )
377  , NonConstArgIsResult( false )
378  {}
379  };

381  /** ********************************************************************************************
382  * Info struct for compiling automatic type casts. Such automatic cast is tried to be inserted
383  * into the expression program by the compiler in exactly two occasions:
384  *
385  * - With binary operators with a combination of types that no plug-in claimed responsibility
386  * for.
387  * - With conditional operator <c>Q ? T : F</c>, if different types for \c T and \c F were
388  * given.
389  *
390  * Which of the two scenarios a cast is needed for is provided with field #BinaryOp.
391  *
392  * Note that the built-in compiler plug-in \alib{expressions::plugins,AutoCast} ignores
393  * this field even if a binary operator is given. It just always tries to match both types
394  * to the 'major' one, in the "hope", that for two same types an operator is available
395  * when the search for such is repeated after a successful cast.<br>
396  *
397  * In contrast to this, a custom plug-in may choose to cast both values to a joint one or to
398  * a combination of types that it provides an operator for!
399  *
400  * Compile-time optimization is supported with auto-casts the same as with other compilation
401  * mechanics.
402  * Information about whether the arguments are constants is separately given for \e lhs and
403  * \e rhs with fields #LhsIsConst and #RhsIsConst.
404  *
405  * Hence, if a plug-in leaves the parent field
406  * \alib{expressions::CompilerPlugin::CompilationInfo,Callback} and/or field #CallbackRhs nulled,
407  * but stores a constant result value in
408  * \alib{expressions::CompilerPlugin::CompilationInfo,TypeOrValue} and/or #TypeOrValueRhs
409  * (by assigning the casted value), then this is detected by the compiler and instead of
410  * inserting a cast function call, the original constant value is replaced with the returned
411  * constant value(s).
412  *
413  * If a cast function is compiled (returned with this struct), and the resulting program
414  * should be duly \e "decompilable", then along with the callback information, a
415  * compilable expression function name has to be returned in field
416  * #CastExpressionFunctionNameLhs, respectively #CastExpressionFunctionNameRhs.
417  * For further information on this topic see
418  * \ref alib_expressions_details_optimizations_norm "11.5.6 Optimized Expression Strings"
419  **********************************************************************************************/
420  struct CIBinaryAutoCast : public CompilationInfo
421  {
422  /**
423  * The operator that the cast is required for. If this is \e nulled then
424  * the request is made for conditional operator <c>Q ? T : F</c>. In this case, the
425  * requirement is to cast both sides to the very same type - otherwise, the
426  * conditional operator does not compile!
427  */
428  String& BinaryOp;
429 
430  /// Input: denotes if lhs argument is constant value.
431  bool LhsIsConst;
432 
433  /// Input: denotes if rhs argument is constant value.
434  bool RhsIsConst;
435 
436  /**
437  * Output: Native C++ callback function to cast the first type with.
438  * \note
439  * The optional callback function for casting the left-hand side type is returned with
440  * inherited field \alib{expressions::CompilerPlugin::CompilationInfo,Callback}.
441  */
442  CallbackDecl CallbackRhs;
443 
444  /**
445  * Output: Specifies the return type of #CallbackRhs, respectively, as the name indicates,
446  * the result value in case of constant a result.<br>
447  * In case of constant compile-time values, it might be necessary to allocate
448  * compile-time memory for the values. For this, field #CompileTimeScope is to be used.
449  * \note
450  * The optional return type and value casting the left-hand side argument is returned with
451  * inherited field \alib{expressions::CompilerPlugin::CompilationInfo,TypeOrValue}.
452  */
453  Box TypeOrValueRhs;
454 
455  /**
456  * This is the name of the left-hand side cast function that is inserted when an expression
457  * with an auto-cast functions is \e decompiled to generate compilable, optimized expression
458  * strings.
459  */
460  String CastExpressionFunctionNameLhs;
461 
462  /**
463  * This is the name of the right-hand side cast function that is inserted when an expression
464  * with an auto-cast functions is \e decompiled to generate compilable, optimized expression
465  * strings.
466  */
467  String CastExpressionFunctionNameRhs;
468 
469  #if ALIB_DEBUG
470  /**
471  * Output: The C++ name of the callback function. This field is available only in debug
472  * compilations of the library. Hence, setting it must be performed with preprocessor
473  * conditionals.
474  */
475  const char* DbgCallBackNameRhs;
476  #endif
477 
478  /**
479  * Constructor.
480  * @param scope Passed to parent.
481  * @param binaryOp Stored in #BinaryOp.
482  * @param lhsIsConst Stored in LhsIsConst.
483  * @param rhsIsConst Stored in RhsIsConst.
484  */
485  CIBinaryAutoCast( Scope& scope, String& binaryOp, bool lhsIsConst, bool rhsIsConst )
486  : CompilationInfo ( scope )
487  , BinaryOp ( binaryOp )
488  , LhsIsConst ( lhsIsConst )
489  , RhsIsConst ( rhsIsConst )
490  , CallbackRhs ( nullptr )
491  , TypeOrValueRhs ( nullptr )
492  {
493  }
494  };

513  /** ********************************************************************************************
514  * Used to compile identifiers (parameterless functions ) and functions parsed from
515  * expression strings.
516  *
517  * The function name is given as an in/out parameter. Implementations might (should) choose to
518  * - on the one hand allow abbreviations and letter case insensitive identifier recognitions, and
519  * - consequently return 'corrected' identifier names if an identifier name matched and the
520  * element got compiled.
521  *
522  * Such corrected names will appear in the normalized expression strings returned by
523  * \alib{expressions,Expression::GetNormalizedString}, in case flag
524  * \alib{expressions,Normalization::ReplaceFunctionNames} is set in field
525  * \alib{expressions,Compiler::CfgNormalization}.
526  *
527  * On success, this method has to provide a (native C++) callback function that accepts
528  * start and end iterators of boxed arguments which are of the same time as proposed by
529  * the corresponding iterators found in parameter \p{ciFunction}, along with the return type
530  * of that function.
531  *
532  * Alternatively, if a constant identifier is compiled or if all parameters are known to be
533  * constant at compile time, a constant value might be returned.
534  * For details of the input and output parameters of the function, see struct
535  * \alib{expressions::CompilerPlugin,CIFunction}.
536  *
537  * @param[in,out] ciFunction The compilation info struct.
538  * @return Implementations have to return \c true if the given info struct was filled, or in
539  * other words, if the plug-in chose to compile the <em>AST</em>-node.<br>
540  * This default implementation returns \c false to indicate that no compilation
541  * was done.
542  **********************************************************************************************/
543  virtual bool TryCompilation( CIFunction& ciFunction ) { (void) ciFunction; return false; }
544 
545  /** ********************************************************************************************
546  * Used to compile unary operators parsed from expressions.
547  *
548  * On success, this method has to provide a native C++ callback function together with
549  * a \ref alib_expressions_prereq_sb "sample box" that specifies its return type.
550  * Both are to be stored in output parameter \p{result}.
551  *
552  * Alternatively, if field \c %ArgIsConst of the given compilation info struct is \c true,
553  * a constant value might be returned.
554  * For details of the input and output parameters of the function, see struct
555  * \alib{expressions::CompilerPlugin,CIUnaryOp}.
556  *
557  * The implementation might allow alias operators. If such alias is found, the used original
558  * operator should be returned in/out parameter \p{operation}.
559  *
560  * Such "corrected" operators will appear in the normalized expression strings returned by
561  * \alib{expressions,Expression::GetNormalizedString}, in case flag
562  * \alib{expressions,Normalization::ReplaceAliasOperators} is set in field
563  * \alib{expressions,Compiler::CfgNormalization}.
564  *
565  * @param[in,out] ciUnaryOp The compilation info struct.
566  * @return Implementations have to return \c true if the given info struct was filled, or in
567  * other words, if the plug-in chose to compile the <em>AST</em>-node.<br>
568  * This default implementation returns \c false to indicate that no compilation
569  * was done.
570  **********************************************************************************************/
571  virtual bool TryCompilation( CIUnaryOp& ciUnaryOp ) { (void) ciUnaryOp; return false; }

573  /** ********************************************************************************************
574  * Used to compile binary operators parsed from expressions.
575  *
576  * On success, this method has to provide a native C++ callback function together with
577  * a \ref alib_expressions_prereq_sb "sample box" that specifies its return type.
578  * Both are to be stored in output parameter \p{result}.
579  *
580  * Alternatively, depending on fields \c %LhsIsConst and \c %RhsIsConst of the given compilation
581  * info struct, a constant value might be returned or, as a third alternative, field
582  * \c %NonConstArgIsResult may be set to \c true, if an "identity" operation is detected
583  * For details of the input and output parameters of the function, see struct
584  * \alib{expressions::CompilerPlugin,CIBinaryOp}.
585  *
586  * The implementation might allow alias operators. If such alias is found, the used original
587  * operator should be returned in/out parameter \p{operation}.
588  *
589  * Such "corrected" operators will appear in the normalized expression strings returned by
590  * \alib{expressions,Expression::GetNormalizedString}, in case flag
591  * \alib{expressions,Normalization::ReplaceAliasOperators} is set in field
592  * \alib{expressions,Compiler::CfgNormalization}.
593  *
594  *
595  * @param[in,out] ciBinaryOp The compilation info struct.
596  * @return Implementations have to return \c true if the given info struct was filled, or in
