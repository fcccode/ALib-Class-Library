Class Description

template<typename TApplicable, typename TChar, typename TEnableIf = void>
struct aworx::lib::strings::T_Apply< TApplicable, TChar, TEnableIf >

This is a template struct which answers the question "Can an object (value) of type TApplicable be passed to method AString::Apply? By default, the answer is no. In particular, the default implementation of the struct inherits std::false_type.

To make a custom type "applicable", this struct needs to be specialized for the custom type with inheriting from std::true_type and with implementing method Apply.

This concept of specializing this struct "once" and then being able to pass custom types to objects of class AString has a central role for using ALib in a non intrusive way: Methods of ALib classes (or third-party code that uses ALib) may offer APIs that can be fed with custom types without the prior need of conversion.

A list of built-in specializations for standard C++ and ALib types is given and documented here.

To restrict the need for specializations of this method to a single variant of the external type, template type TApplicable to specialize will always be a the plain custom type. No pointer, const, etc. versions need to be implemented.

The implementation of method Apply may do 'whatever' is wanted with the AString provided in parameter target. For string types, simple append operations are predefined. More complex operations format the string, convert integer and floating point numbers, etc.

For basic user defined string types that get adopted to ALib string system using a specialization of struct T_String, no specialization of this struct is needed, because method AString::Apply will detect that and append the custom string.

The second template parameter TEnableIf may be used to perform templated specializations based on a condition evaluated with std::enable_if (or similar tmp mechanics).

Built-in specializations of this function for certain types exist, as listed in reference documentation of class AString. Some types exists only for the reason to be "applicable" to AString! As a sample, class Format::Field allows to align strings when they are applied. The invocation is simple and nicely readable:

AString centered;
centered << '*' << Format::Field( "Hello", 15, Alignment::Center ) << '*';
cout << centered << endl;

which produces:

* Hello *

The next sample implements this function for ALib class DateTime to print out a formatted date:

#include "alib/alib.hpp"
#include "alib/strings/numberformat.hpp"
// get support for ostream operator<<() on String objects
#include "alib/compatibility/std_string.hpp"
#include <iostream>
#include <iomanip>
// Partial template function implementation of struct T_Apply for objects of type class DateTime
// (has to be declared in namespace aworx::lib::strings)
namespace aworx { namespace lib { namespace strings {
// We have to specialize the helper struct T_Apply.
template<> struct T_Apply<aworx::lib::time::DateTime, character> : public std::true_type
{
// And with it, method Apply
static inline integer Apply( AString& target, const aworx::lib::time::DateTime& ticks )
{
time::CalendarDateTime calendarTime;
calendarTime.Set( ticks, Timezone::UTC );
calendarTime.Format( ASTR("yyyy-MM-dd HH:mm"), target );
return 16; // The exact number is not too relevant. But has to be > 0 if something was written!
}
};
}}} // closing namespace

With this definition included, user code might now do this:

AString sample( "Today is: " );
sample << DateTime();
cout << sample << endl;

The output will be something like this:

Today is: 2018-05-12 13:53

Default implementation of Apply. This method must never be invoked and therefore throws an assertion if it is. Prior to do so, a static_assert will prevent code to apply types to an object of class AString that are not applicable. However, it might happen that this compile-time assertion fails to be activated in some tricky situations. In this case, the runtime assert will step in.

Implementations for specialized versions of this struct might do whatever they are supposed to with target. Actions are not restricted to appending something.

Note
Some more complex applicable types might want to return extended information by setting own members during the application process. To achieve this, the corresponding implementations of this method have to cast the given const reference to a non-const reference. This is considered a good practice, as the const specifier most probably was added by the TMP operations of method AString::Apply anyhow - only for the reason to have one single template specialization per type.
Parameters
targetThe object of type AString that will have object src applied.
srcThe source object of arbitrary type will get applied to AString target.
Returns
The number of characters that got appended should be returned ('should' here means that at least a non zero, positive value has to be returned) and -1 if the object src which is to be applied represented a nulled object. Returning -1 causes the AString to be nulled if the apply operation was invoked through AString assignment operator.
If nothing is was appended, this method must return 0.
