1131 /**
1132  * This \ref alib_namespace_boxing_interfaces "ALib boxing interface" is used to
1133  * create parsable expression "literals" from constant values of custom type stored in boxes.
1134  * The interface is used by the library in the following circumstances:
1135  * - If custom identifiers, functions or operator callback functions return a custom type.
1136  * - If such types can be constants and are announced to the compiler as such.
1137  * - If method \alib{expressions,Expression::GetOptimizedString} is called.
1138  * - And, not necessarily: if such string is to be used as input to compiling expressions
1139  *
1140  * The challenge here is to convert constants of custom types back into a normalized, human readable
1141  * but also compilable expression string.
1142  * As the expression syntax only defines the built-in literal types
1143  * \alib{expressions::Types,Integer}, \alib{expressions::Types,Float} and
1144  * \alib{expressions::Types,String}, the constants have to be created using either appropriate
1145  * custom identifiers or "constructor functions" that have to be provided in addition along with the
1146  * as soon as the conditions above are met. These identifiers and constructor functions in turn
1147  * need to be compile-time evaluatable to ensure that recompiling the optimized string results
1148  * to constants.
1149  *
1150  * So far, this probably sounded a little complicated, but a step-by step sample taken from
1151  * built-in date/time functionality demonstrates the use quite well:
1152  *
1153  * 1. %Compiler plug-in \alib{expressions::plugins,DateAndTime} introduces \alib class
1154  * \alib{time,TimePointBase::Duration} to expressions. This is for example done by defining identifiers as follows:
1155  *
1156  * \snippet "plugins/dateandtime.cpp" DOX_ALIB_EXPR_IToLiteral_1
1157  *
1158  * The "constructor functions" are declared to be \alib{expressions::plugins,Calculus::CTI,compile-time invokable}
1159  * and return are a constant at compile-time if their input parameter is constant.
1160  * When the program - that may due to optimization not contain the identifiers any more - becomes
1161  * de-compiled, these constants have to be written to the normalized expression string in a way that
1162  * corresponding constant values of type \b %Duration are expressed.
1163  *
1164  * To perform exactly this task, this boxing interface class is used.
