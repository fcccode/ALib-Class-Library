22 /** ************************************************************************************************
23  * Implementation of the default parser of module \alibmod_nolink_expressions.
24  *
25  * This internal class is not too well documented. Nevertheless, it is designed to be able
26  * to tweak its behaviour slightly and in case of need, even derive and use a custom parser
27  * class. For doing so, please consult the source code of this class.
28  *
29  * It is possible to define scannable custom unary and binary operators. Definitions of binary
30  * operators include a "precedence value" that allows to align them with the built-in types.
31  * Also, built-in operators can be \em removed if wanted.
32  *
33  * \see
34  * Besides this class, a parser based on \c boost::spirit is available as an alternative.
35  * This is documented with appendix manual section
36  * \ref alib_expressions_appendix_boostspirit "A.3 Using Built-In Alternative Parser Based On boost::spirit".
37  **************************************************************************************************/
