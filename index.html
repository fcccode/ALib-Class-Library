The ALib C++ Class Library

ALib is a general purpose C++ class library. Its main characteristics and design goals are:

  • It is free software.
  • The minimum C++ language standard is C++ 11, selective support for more modern C++ versions included.
  • Modularization: Possible selective use of only parts of the library.
  • Extensive documentation (Reference documentation, tutorials, sample code).
  • Non-intrusiveness: Designed to keep user code as independent from ALib as possible.
  • Developed and steadily tested under GNU/Linux, Windows OS and Mac OS with compilers GNU, Clang and Microsoft Visual C++.

Documentation

The following documentation is provided.

  1. A Reference documentation of all entities of the library (100% complete) is provided. For this, source code documentation tool Doxygen is used.
  2. A Programmer's manual is available for each ALib Module
  3. Page Change Log provides a detailed version history.
  4. Project setup is documented with IDE/Project Setup of ALib for C++.
  5. A reference to C++ macros and compilation symbols is given with ALib and the C++ Preprocessor.
  6. For module ALox, a separated website exists. This includes plenty of ALox specific documentation, a lengthy tutorial, definitions of configuration/environment variables and more.
  7. A lot of sample code is implemented as unit tests and integrated as source and output text snippets into the documentation. (As for example shown here).
Note
The Programmer's Manuals for each module is found embedded in the reference documentation, namely with the documentation of the corresponding C++ namespace entity.
As an example, see documentation of ALib Expressions.

C# and Java Versions Of ALib

Parts of ALib has sibling incarnations in programming languages C# and JAVA. Historically, ALib aimed to be a cross platform/cross language library. This goal was (mostly) dropped in favour to be able to independently develop ALib for C++.

ALib for C# and ALib for Java are included in and distributed with the cross platform ALox Logging Library.

For C++, since May 2018 release, the situation was inverted: Instead of having ALib bundled with ALox, the logging library is now a module of ALib.
