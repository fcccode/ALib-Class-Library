23 /** ************************************************************************************************
24  * Base class for \alib built-in formatters.
25  * This class implements abstract method
26  * \alib{strings::format,FormatterBase::format,Formatter::format} and introduces a set of
27  * new abstract methods that have to be implemented by descendants.
28  *
29  * The documentation of the class is split into two sections: The first addresses users of
30  * descendant classes, the second provides information on how to extend this class to implement
31  * a custom formatter.
32  *
33  * ## Documentation For Users Of Derived Classes ##
34  *
35  * Derived formatter types will set attributes in fields #DefaultNumberFormat and
36  * #AlternativeNumberFormat to reflect the default behavior of their formatting syntax.<br>
37  * Users of derived formatters may modify these attributes prior to performing a format operation
38  * to change those defaults. While this leads to a deviation of the formatting standard, it may be
39  * used instead of providing corresponding syntactic information within each and every replacement
40  * field of the format strings. Some modifications may not even be possible with just using
41  * the format specification.
42  *
43  *
44  * ## Documentation For Implementing Derived, Custom Formatters ##
45  *
46  * Derived types need to set default values for attributes in fields
47  * #DefaultNumberFormat and #AlternativeNumberFormat within their constructor once - according
48  * to defaults specified by the formatting syntax.
49  * This should not be repeated per format operation. This way users of the type are allowed
50  * to change such default setting (even if they this may the formatter deviates from the standard
51  * it implements).
52  *
53  * All fields of this class named with prefix \c pha (for example, #phaAlignment) together comprise
54  * the set of formatting attributes which can be modified by placeholder semantics of the format string.
55  * If further attributes are needed, derived types are free to define custom members to store such
56  * attributes. Implementations provided with \alib give such extended attributes the prefix
57  * \c phaExt.
58  *
59  * When parsing a placeholder of a format string, abstract method #parsePlaceholder
60  * may set field #phaFormatSpec to reflect a format-specific portion the placeholder string.
61  * If it does, it will be checked if the argument supports
62  * <b>%ALib %Boxing</b> interface \ref aworx::lib::strings::boxing::IFormat "IFormat"
63  * and if so, the boxing interface is invoked with this string as parameter.
64  * If the argument does not support the interface, method #parseStdFormatSpec is invoked to
65  * now parse this portion of the placeholder string in a default way.<br>
66  * This concept allows customized format specifications for custom argument types! As an example,
67  * a format specification for date/time argument types may support a custom format string
68  * like \c "yyyy-MM-dd HH:mm:ss".
69  * \note
70  * This concept is implemented with class
71  * \alib{strings::format,FormatterPythonStyleBase,FormatterPythonStyle} as the
72  * "Python format mini language" supports such custom format specifications. Class
73  * \alib{strings::format,FormatterJavaStyleBase,FormatterJavaStyle} does \b not support
74  * this mechanism.
75  *
76  * The following describes the formatting process in detail (the implementation of method #format)
77  * and this way helps to understand what is required from the implementation of the abstract methods:
78  *
79  * 1. A newline character check is made and formatting is aborted if one was found
80  * (as proposed by contract with parent class).
81  *
82  * 2. Method parameters are stored in fields #targetString, #formatString, #arguments and
83  * #argOffset. This way, the parameters can be accessed from any implemented method without
84  * the need of passing them as parameters once more.<br>
85  * In addition, field #parser is initialized. This \b Substring is used to parse
86  * the format string. Parsed portions will be consumed from the front of the string.
87  * Finally fields #argsConsumed and #nextAutoIdx are initialized to value \c 0.
88  *
89  * 3. <b>Start of the loop</b> to find and process placeholders in the format string.
90  *
91  * 4. Abstract method #findPlaceholder is invoked. If this fails (no further placeholder was found)
92  * parsing stops. If, and only if, a placeholder was found before, the rest of the string
93  * in #parser is written and abstract method #replaceEscapeSequences is invoked for this rest.
94  *
95  * 5. The portion of the format string before the placeholder position is written and abstract
96  * method #replaceEscapeSequences is invoked on this portion (if not empty).
97  *
98  * 6. Method #resetPHAs is invoked, to reset the attributes that will be parsed in the next
99  * step. The values that are set are implementation specific and need to reflect the
100  * default formatting options if no specific options are given in the format string.
101  *
102  * 7. Abstract Method #parsePlaceholder is invoked to parse and consume tokens from string #parser
103  * and while doing this, to store the parsed format attributes in the fields with
104  * name prefix \c pha (or extended attributes of a derived formatter type).<br>
105  * If an argument (positional) index is found during parsing, then method
106  * #setArgument is to be invoked by abstract method \b %parsePlaceholder providing that index.<br>
107  * If the format syntax of the formatter contains a separated format specification string
108  * (a substring of the placeholder string), then the method may store such format
109  * substring in field
110  * \ref aworx::lib::strings::format::FormatterStdImpl::phaFormatSpec "phaFormatSpec".
111  *
112  * 8. Next, it is checked if an argument was set by \b %parsePlaceholder. If not, #setArgument
113  * is invoked providing \c -1 for the index to indicate auto-indexing.
114  * \note
115  * If auto-indexing should be implemented differently than done with default method
116  * #setArgument, then a custom formatter might either override the method or,
117  * in the case no index is given in the format string, just
118  * set fields
119  * \ref aworx::lib::strings::format::FormatterStdImpl::phaArgument "phaArgument" and
120  * \ref aworx::lib::strings::format::FormatterStdImpl::phaArgumentIdx "phaArgumentIdx"
121  * already in \b %parsePlaceholder according to its own strategy
122  *
123  * 9. Method #preAndPostProcess is invoked with parameter \p{startIdx} equalling \c -1
124  * (indicating pre-processing). This allows for example to insert tab fill-characters
125  * (tab stops) prior to writing the contents of the field.
126  *
127  * 10. Method #writeCustomFormat is invoked. This allows derived formatters to write arguments in a
128  * custom way. If the method returns \c true, the loop is continued ( &rarr; Step 4.). <br>
129  * The default implementation checks if a format specification was stored in #phaFormatSpec
130  * and if yes, if boxing interface
131  * \ref aworx::lib::strings::boxing::IFormat "IFormat" is defined for #phaArgument.
132  * Un this case, the interface is invoked and \c true returned.
133  *
134  * 11. Again, if a format specification was stored in
135  * \ref aworx::lib::strings::format::FormatterStdImpl::phaFormatSpec "phaFormatSpec"
136  * method #parseStdFormatSpec is invoked which needs to set further attributes
137  * in the \b %Placeholder object according to to the standard format specification of the formatter.
138  *
139  * 12. Now, as all fields that represent formatting attributes (fields with prefix \c pha) are well
140  * set (or kept with their defaulted value), method #checkStdFieldAgainstArgument is invoked.
141  * This method is virtual but not abstract. Its default implementation checks the
142  * placeholder attributes against the provided argument type and raises an error if the
143  * argument does not fit to the placeholder format specification.
144  *
145  * 13. Method #writeStdArgument is invoked. This method is virtual but not abstract.
146  * Its default implementation writes the argument value formatted according to the attribute
147  * fields.
148  *
149  * 14. Finally #preAndPostProcess is invoked with parameter \p{startIdx} pointing to the first
150  * character in #targetString of the argument written.
151  * Here, actions like case conversion might be done on the field written.
152  *
153  * 14. End of loop ( &rarr; Step 4.)
154  *
155  *
156  * @tparam TChar The character type. Implementations for \c char and \c wchar_t are provided.
157  **************************************************************************************************/

446  *

448  * If \p{target} is given, this indicates an "intermediate phase": The argument has been
449  * written, but no alignment or cutting has been done, yet. This phase should usually
450  * be ignored, but is for example important for search and replacement actions.
451  * If a field has a custom format implementation (e.g. time and date values), then
452  * the intermediate phase is never called.
453  *

456  * derived classes might do some more complicated parsing of parts of the placeholder

458  * once, while the finally parsed commands are only conditionally executed depending
459  * if executed as pre or post phase.

464  * indicates intermediate phase.
465  * @return \c false, if the placeholder should be skipped (nothing is written for it).
466  * \c true otherwise.
467  ******************************************************************************************/
