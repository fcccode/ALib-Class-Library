34 /** ************************************************************************************************
35  * As explained in detail in the documentation of namespace \ref #aworx::lib::time, \alib
36  * supports a steady time model with class \alib{time,Ticks} and one representing the
37  * system clock with class \alib{time,DateTime}. Both types share this template class
38  * as their generic base.
39  *
40  * The common features that this class provides to its descendants are:
41  * - Type definition #TTimePoint used to store a point in time.
42  * - Method #NativeValue, which exposes the internal value of type #TTimePoint which is of C++
43  * standard type \c std::chrono::time_point.
44  * - Inner class \alib{time::TimePointBase,Duration} that represents the difference type of this
45  * class.
46  * - Static method #Now which create an instance representing the current point in time.
47  * - Various overloaded arithmetic and comparison operators.
48  * - Methods #Age and #Since.
49  *
50  * \note
51  * The resolution and accuracy of the values is platform dependent. Especially nanoseconds are
52  * deemed to be not accurate and above 100 ns (this was written and fact in 2013, reviewed 2018).
53  *
54  * \attention
55  * The dates storable in objects of this class is limited to a certain time range. In the
56  * current GNU/Linux and Windows implementations the range is roughly <c>+-292.27</c> years before
57  * and after the point in time that the system that the software is running on was initialized
58  * (booted). This value results from dividing <c>2‚Å∂63 bit</c> by the number of nanoseconds of
59  * one year of 365.25 days and the fact that internal, system dependent ticks counters are reset
60  * to \c 0 with the boot of a system.
61  *
62  * @tparam TClock The type of clock to use. This will be
63  * - \c std::chrono::system_clock with descendant class \alib{time,DateTime} and
64  * - \c std::chrono::steady_clock with descendant class \alib{time,Ticks}.
65  * @tparam TDerived The derived type itself, hence either \alib{time,DateTime} or
66  * \alib{time,Ticks}. This template parameter is needed to define the result
67  * type of various methods and operators.
68  **************************************************************************************************/

465  /** ************************************************************************************
466  * Returns 1 divided by internal value in seconds, hence the number of Hertz that this
467  * object represents when interpreted as a time span.
468  *
469  * @param qtyFractionalDigits Number of digits that the return value will be rounded to.
470  * Defaults to -1 which means no rounding.
471  * @return double value representing the frequency in hertz.
472  **************************************************************************************/
473  double InHertz( int qtyFractionalDigits= -1 ) const
474  {
475  // check
476  if ( span.count() == 0)
477  return 0.0;
478 
479  // calc hertz
480  double hz= 1000000000.0 / InNanoseconds();
481 
482  // no rounding? that's it
483  if ( qtyFractionalDigits < 0 )
484  return hz;
485 
486  // round
487  double mag= pow( 10, qtyFractionalDigits );
488  return static_cast<int>( hz * mag + ( hz < 0 ? -0.5 : 0.5 ) )
489  / mag;
490  }
