139 /** ********************************************************************************************
140  * This is a specializable struct which defaults the question whether a given type
141  * can be used to construct an \alib{strings,StringBase,String} to \c false.
142  *
143  * Specializations of this struct inherit from \c std::true_type, to define a type to be a usable
144  * constructor parameter type of \b %String.
145  *
146  * Hence, this template struct supports converting 'external' <em>user defined</em> to
147  * \alib{strings,StringBase,String}. It has a central role for using \alib in a
148  * \ref aworx::lib "non intrusive" way, by allowing to pass external string types just as
149  * the are when invoking \alib functions and methods that expect parameters of type
150  * <c>const %String&</c>. Due to the implicit constructor invocation of C++
151  * and the template meta programming in constructor of class \b %String, custom types
152  * (that specialize this struct) may be passed "as is" to such methods.
153  *
154  * Methods #Buffer and #Length of partially specialized versions of this struct are invoked
155  * by constructor \alib{strings::StringBase,StringBase(const TStringLike&)}
156  * to receive a pointer to the character buffer and the length of the string.
157  *
158  * The following sample demonstrates the use of this technique:
159  *
160  * \snippet "DOX_ALIB_TOAS.cpp" DOX_ALIB_TOAS
161  * The output will be:
162  * \snippet "DOX_ALIB_TOAS.txt" OUTPUT
163  *
164  * \note
165  * As shown in the sample, if a specialized version of this struct for a custom type exists, such
166  * type becomes automatically also an "applicable" type to class #AString, because method
167  * \b %AString::Apply checks for the existence of a specialization of this struct likewise
168  * the constructor of class \b %String does.
169  *
170  * A compile-time assertion will be raised when a String is constructed with a type that has
171  * no specialization of this struct.
172  *
173  * @tparam TStringLike The type for which string construction compatibility is declared.
174  * @tparam TChar The character type of the strings that can be constructed with
175  * \p{TStringLike}.
176  * **********************************************************************************************/
177 template<typename TStringLike, typename TChar>
178 struct T_String : public std::false_type
179 {
180  /** ********************************************************************************************
181  * In specialized versions, this method has to return a pointer to the buffer of the given
182  * object of type \p{TStringLike}.
183  *
184  * This default implementation makes some static assertions to detect usage of an unsupported
185  * type (a type that is not allowed to construct class \b %String with).
186  *
187  * @returns Has to return a pointer to the buffer of object of type T, respectively its length.
188  **********************************************************************************************/
189  static inline constexpr
190  const TChar* Buffer( const TStringLike& )
191  {
192  // should never happen, because of static assert in string constructor
193  return nullptr;
194  }
195 
196  /** ********************************************************************************************
197  * In specialized versions, this method has to return the length of the buffer of the given
198  * object of type \p{TStringLike}.
199  *
200  * @returns Has to return the length of the string.
201  **********************************************************************************************/
202  static inline
203  integer Length( const TStringLike& )
204  {
205  return (integer) 0;
206  }

255 /** ************************************************************************************************
256  * This class is the base class of all \alib string classes. It represents a character string
257  * whose data is allocated outside the scope of objects of this class.
258  *
259  * Once constructed, objects
260  * of this class are immutable. This means, there is no interface to change their buffer or length
261  * during their lifetime, which normally is rather volatile.
262  *
263  * The buffer represented might be zero-terminated or not. This is dependent on the data it was
264  * constructed from. The buffer must not be changed and might even reside in read-only memory.
265  *
266  * <b>Templated Construction</b><br>
267  * What makes this class very flexible, is the constructor variant
268  * \alib{strings,StringBase::StringBase(const TStringLike&)}.
269  * This template method internally uses <em>template meta programming</em> to detect known types
270  * and, and to convert them to constant references of those. They are then passed to template
271  * methods \alib{strings,T_String} that simply return a pointer to the external types' buffer
272  * and length. This way, objects of this class can be implicitly constructed from just anything that
273  * 'smells' like a string.<br>
274  * For more information on how to make %String support to implicitly construct from user defined types,
275  * see namespace template function
276  * \ref aworx::lib::strings::T_String "T_String".
277  *
278  * This class provides compiler defined copy and move constructors and assignment operators.
279  * Once and object is constructed, methods #Buffer and #Length allow read access to the contents.
280  *
281  * <b>Null-State</b><br> \anchor CPP_STRINGS_AS_NULLSTATE
282  * Objects of this class can be \e nulled which means that it is a difference whether they are
283  * representing an empty string or a \c nullptr. As objects are immutable, this is decided
284  * on construction: once a %String is nulled, it will remain nulled.
285  * In other words, it makes a difference if an %String is constructed using
286  * - <em>%String()</em>, which creates a \e nulled object, method #IsNull will give \c true, or
287  * - <em>%String("")</em>, which creates an empty object. Method #IsNull will give \c false while
288  * method #IsEmpty will give \c true
289  *
290  * <b>Non-checking Method Variants</b><br> \anchor CPP_STRINGS_AS_NC
291  * Some of the provided methods are templated with boolean parameter <em>TCheck</em> which allow
292  * to invoke a faster and shorter version of the same method. For more information see
293  * \ref alib_strings_details_nonchecking in the namespace documentation.
294  *
295  * \note Almost all methods of this class are declared <em>inline</em>.
296  *
297  * <p>
298  * \note The immutable nature of %String is lifted by derived types. While class
299  * \alib{strings,SubstringBase,Substring} allows to change the start and
300  * length of the string represented, class
301  * \alib{strings,AStringBase,AString} holds a copy of the data and allows to
302  * modify the contents. Field #buffer of this class is declared
303  * as an anonymous union of a <em>const char*</em> and a <em>char*</em> but the latter is
304  * not exposed. It might be exposed by derived classes (and is by class \p{%AString}).
305  *
306  * <p>
307  * \note For an introduction into the \alib string classes see
308  * \ref aworx::lib::strings "namespace strings"
309  *
310  * @tparam TChar The character type. Implementations for \c char and \c wchar_t are provided
311  * with type definitions \ref aworx::NString and \ref aworx::WString.
312  **************************************************************************************************/
