aworx::lib::boxing::thirdparty::std Namespace Reference

Programmer's Manual

Implements ALib Boxing for string types of the standard library. On boxing, the string types are mapped to boxing arrays of the corresponding character types.

Note
The boxing definitions described here are available through header file
"alib/compatiblity/std_string.hpp".

The following specializations are made:

  • Type std::string is boxed as char[]
  • Type std::wstring is boxed as wchar_[]
  • If wchar_t is 2 bytes wide, then std::u32string is boxed as aworx::strangeChar[]
  • If wchar_t is 4 bytes wide, then std::u16string is boxed as aworx::strangeChar[]

The same mapping applies for pointer types of named std types.

If languages standard C++ 17 is in place, also the corresponding "view" classes (e.g. std::string_view) are boxed likewise.

All types are unboxable als values.

Note
Namespace function InitStdString is to be invoked on bootstrap to complete the support of std::string-type boxing.

void InitStdString ( )
inline

Initializes ALib Boxing system in respect to boxing std::string-types.

This method is not automatically invoked with ALib::Init, because the support for boxing std::string-types is optional and due to the inclusion of header "alib/compatibility/std_string.hpp".

In general boxing of std::string-types works well without the one-time invocation of this function at the bootstrap of a process. What this method currently adds is interface IApply for boxed type BoxedAs<std::string> and similar std string types. This is only needed in some particular use cases.
